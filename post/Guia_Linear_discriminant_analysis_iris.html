Para conseguir una estimacin ms certera, y antes de recurrir al conjunto de test, se pueden emplear estrategias de validacin basadas en resampling. Cada uno funciona internamente de forma distinta, pero todos ellos se basan en la idea de ajustar y evaluar el modelo mltiples veces con distintos subconjuntos creados a partir de los datos de entrenamiento, obteniendo en cada repeticin una estimacin del error. El promedio de todas las estimaciones tiende a converger en el valor real del error de test.

Caret incorpora los mtodos boot, boot632, optimism_boot, boot_all, cv, repeatedcv, LOOCV, LGOCV.

En este caso se recurre a validacin cruzada repetida como mtodo de validacin.

number = Numero de particiones = 3

repeats = Numero de repeticiones = 3

Una validacin cruzada con 3 particiones y 3 repeticiones, implica ajustar y evaluar el modelo 3 x 3 = 9 veces, cada vez con una particin distinta, ms un ltimo ajuste con todos los datos de entrenamiento para crear el modelo final.

El mtodo predeterminado para optimizar los parmetros de ajuste trains usar una grid search. Este enfoque suele ser efectivo pero, en los casos en que hay muchos parmetros de ajuste, puede ser ineficiente. Hay varios modelos en los que esto puede ser beneficioso para encontrar valores razonables de los parmetros de ajuste en un tiempo relativamente corto. Sin embargo, hay algunos modelos donde la eficiencia en un campo de bsqueda pequeo puede cancelar otras optimizaciones. Para usar la bsqueda aleatoria, hay otra opcin disponible en trainControl llamadas search. Los valores posibles de este argumento son "grid"y "random". Los modelos incorporados en caret contienen cdigo para generar combinaciones de parmetros de ajuste aleatorio. El nmero total de combinaciones nicas se especifica mediante la opcin tuneLength.

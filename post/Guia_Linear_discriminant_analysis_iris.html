Para conseguir una estimación más certera, y antes de recurrir al conjunto de test, se pueden emplear estrategias de validación basadas en resampling. Cada uno funciona internamente de forma distinta, pero todos ellos se basan en la idea de ajustar y evaluar el modelo múltiples veces con distintos subconjuntos creados a partir de los datos de entrenamiento, obteniendo en cada repetición una estimación del error. El promedio de todas las estimaciones tiende a converger en el valor real del error de test.

Caret incorpora los métodos boot, boot632, optimism_boot, boot_all, cv, repeatedcv, LOOCV, LGOCV.

En este caso se recurre a validación cruzada repetida como método de validación.

number = Numero de particiones = 3

repeats = Numero de repeticiones = 3

Una validación cruzada con 3 particiones y 3 repeticiones, implica ajustar y evaluar el modelo 3 x 3 = 9 veces, cada vez con una partición distinta, más un último ajuste con todos los datos de entrenamiento para crear el modelo final.

El método predeterminado para optimizar los parámetros de ajuste trains usar una grid search. Este enfoque suele ser efectivo pero, en los casos en que hay muchos parámetros de ajuste, puede ser ineficiente. Hay varios modelos en los que esto puede ser beneficioso para encontrar valores razonables de los parámetros de ajuste en un tiempo relativamente corto. Sin embargo, hay algunos modelos donde la eficiencia en un campo de búsqueda pequeño puede cancelar otras optimizaciones. Para usar la búsqueda aleatoria, hay otra opción disponible en trainControl llamadas search. Los valores posibles de este argumento son "grid"y "random". Los modelos incorporados en caret contienen código para generar combinaciones de parámetros de ajuste aleatorio. El número total de combinaciones únicas se especifica mediante la opción tuneLength.

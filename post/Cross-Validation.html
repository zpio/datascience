Cross-Validation (Validación Cruzada)¶

Cross-Validation o validación cruzada, es un método que se emplea para estimar el test error rate de un modelo y así evaluar su capacidad predictiva. También se puede emplear para seleccionar el nivel de flexibilidad adecuado como el grado del polinomio o el número de K-Nearest-Neighbors.

En la ejecución de un modelo al utilizar 100% de sus datos podría generar un resultado que no se generaliza bien a los nuevos datos entrantes. Tambien existe otros escenario, dividir los datos originales en un conjunto de entrenamiento, que generalmente toma alrededor del 70% de los datos y un conjunto de prueba que comprende el resto, el 30%. Se libera el modelo con los datos de entrenamiento y luego usa el conjunto de pruebas para verificar cuál es la salida del modelo en comparación con las respuestas que tienes disponibles.

Sin embargo, este proceso de entrenamiento y prueba aún es algo limitado. En una capacidad, cuando está probando la salida del modelo contra los datos reservados, solo está viendo cuál es el error para esa agrupación exacta de los datos de prueba. En teoría, los datos de la prueba deberían ser representativos de todo el conjunto de datos, pero en la práctica hay casos en los que eso podría no ser cierto. Se debería entrenar el modelo de tal manera que pueda estar seguro de que el error es representativo de todo el conjunto de datos, no solo del segmento específico que obtiene de los datos seleccionados al azar que coloca en el conjunto de prueba.

La validación cruzada es una técnica estadística mediante la cual toma todo su conjunto de datos, lo divide en una serie de pequeños fragmentos de entrenamiento/prueba, evalúa el error para cada fragmento y luego promedia esos errores finales. Este enfoque termina siendo una forma más precisa de evaluar si su enfoque de modelado tiene algún problema que podría estar oculto en varias combinaciones de las partes de entrenamiento y prueba del conjunto de datos.

De hecho, ¡ya hemos realizado una forma de validación cruzada! La simple división 70/30 de entrenamiento/prueba que es un tipo de "validación cruzada". Sin embargo, existen muchas otras técnicas estadísticas de validación cruzada, y con R teniendo su base en el diseño estadístico, puede modelar muchos tipos diferentes de validación cruzada.

Tipos de Data Set¶

  • Training data set: observaciones con las que se genera el modelo.

  • Test data set: observaciones del training data set no empleados en la creación del modelo.

Tipos de errores¶

  • Training error rate: error que comete el modelo al predecir con observaciones del training data set.

  • Test error rate: error que comete el modelo al predecir con observaciones del test data set.

Entonces podemos decir que Cross-Validation o validación cruzada, se emplea para estimar el test error rate de un modelo y así evaluar su capacidad predictiva. También se puede emplear para seleccionar el nivel de flexibilidad adecuado como grado del polinomio o el número de K-Nearest-Neighbors.

Por lo general, en variables cuantitativas, el error se mide mediante mean square error (MSE) o con el RMSE. Cuando se trata de variables cualitativas (clasificación), el error se mide como la proporción de predicciones incorrectas respecto al total de predicciones.

Cross-Validation Simple¶

El método más sencillo consiste en dividir aleatoriamente las observaciones disponibles en dos grupos, uno se emplea para entrenar al modelo y otro para evaluarlo.

LIMITACIONES:

VARIANZA ALTA: La estimación del test error rate es altamente variable porque depende de qué observaciones se incluyan como set de entrenamiento y cuáles como set de validación.

BIAS ALTO: Esto quiere decir que al tener pocos datos para entrenar el modelo, se dispone de menos información, por lo que se reduce su capacidad predictiva. Lo que provocaría sobrestimación del test error.

Al tomar muestras de datos, debe tener cuidado con los cuatro niveles diferentes de sesgo y varianza que podría tener

  1. Bajo sesgo(bias), baja varianza: El mejor de los casos. Las muestras son bastante representativas de la población.
  1. Alto sesgo(bias), baja varianza: Todas las muestras son bastante consistentes, pero no reflejan realmente a la población.
  1. Bajo sesgo(bias), alta varianza: Las muestras varían enormemente en su consistencia, pero algunas pueden ser representativas de la población.
  1. Alto sesgo(bias), alta varianza:Las muestras son un poco más consistentes, pero no es probable que sean representativas de la población.

Cross Validation simple con 100 repeticiones para el calculo del Test error rate

Como se ha comentado previamente, una de las principales desventajas de cross-validation simple es que la estimación de error varía mucho dependiendo de cómo se hayan repartido los datos entre set de entrenamiento y set de validación. A continuación se calcula el test error rate para 100 repeticiones de la validación, repartiendo en cada una de ellas los datos de forma aleatoria.

In [12]:
cv_MSE <- rep(NA,100)

for (i in 1:100) {

  data.samples <- sample(1:nrow(data), nrow(data) * 0.7, replace = FALSE)
                         
  training.data <- data[data.samples, ]
  
  test.data <- data[-data.samples, ]
  
  train.linear <- lm(y ~ x, training.data)
  
  predicciones <- predict(train.linear, test.data)
  
  cv_MSE[i] <- mean((predicciones  - test.data$y)^2)
}

Flexibilidad del modelo mediante Cross_Validation Simpl: elección del grado polinomio del modelo

Introduciendo flexibilidad en el modelo mediante polinomios de mayor grado podría reducir el error de predicción. La validación cruzada nos permite identificar con qué grado de polinomio se consigue el mejor modelo (menor test cv-MSE).

A continuación se ajustan 10 modelos distintos empleando polinomios de grado 1 hasta 10 y se registra el cv-MSE para cada uno. La función poly() permite determinar el grado de polinomio empleado.

In [16]:
cv_MSE <- rep(NA,10)

set.seed(2020)
data.samples <- sample(1:nrow(data), nrow(data) * 0.7, replace = FALSE)
training.data <- data[data.samples, ]
test.data <- data[-data.samples, ]

for (i in 1:10) {

  train.linear <- lm(y ~ poly(x,i), training.data)
  
  predicciones <- predict(train.linear, test.data) 
  
  cv_MSE[i] <- mean((predicciones  - test.data$y)^2)
}

K-Fold Cross-Validation¶

El método K-Fold Cross-Validation es también un proceso iterativo. Consiste en dividir los datos de forma aleatoria en k grupos de aproximadamente el mismo tamaño. k-1 grupos se emplean para entrenar el modelo y uno de los grupos se emplea como test, este proceso se repite k veces utilizando un grupo distinto como test en cada iteración. El proceso genera k estimaciones del test error cuyo promedio se emplea como estimación final.

Observaciones:

El número de iteraciones necesarias viene determinado por el valor k escogido. Por lo general se recomienda un k entre 5 y 10.

La principal ventaja de K-fold CV es que consigue una estimación precisa del test error rate gracias a un mejor Balance entre bias y varianza. En el método K-fold CV los k grupos empleados como entrenamiento son mucho menos solapantes, lo que se traduce en menor varianza al promediar las estimaciones de error.

En R se puede realizar CV de cualquier generalized linear model creado mediante glm() empleando la función cv.glm() del paquete boot. La función glm() engloba diferentes tipos de modelos lineales que se especifican mediante el argumento family. En el caso de regresión lineal no se especifica ningún tipo de familia y es equivalente a emplear la función lm(). Para regresión logística se emplea family=binomial.

La función cv.glm() calcula el error de predicción mediante cross-validation. Si el argumento k no se especifica, se le asigna automáticamente el número de observaciones empleadas para crear el modelo, por lo que se realiza un leave-one-out cross-validation (es decir una 1 observacion para test y el resto para entramiento). La función devuelve una lista con múltiples componentes, el resultado de la validación se almacena dentro del vector delta que contiene la estimación de error con y sin corrección.

La estimación de error del modelo lineal mediante K-fold Cross-Validation k=10 es de 292.51, un valor muy próximo al estimado mediante Validacion Cross-Validation Simple (296.46), pero empleando mucho menos tiempo de computación. Cuando se especifica el número de grupos K a emplear en la validación, la función cv.glm() devuelve dos resultados, uno con corrección de continuidad y otro sin ella.

Si se estudia la influencia de la flexibilidad del modelo, los resultados muestran la misma tendencia que Cross-Validation Simple. Se obtiene una mejora importante empleando una función cuadrática o cubica en comparación al modelo lineal. Para polinomios de grado > 3 la mejora es mínima.

Cross-Validation (Validación Cruzada)¶

Cross-Validation o validacin cruzada, es un mtodo que se emplea para estimar el test error rate de un modelo y as evaluar su capacidad predictiva. Tambin se puede emplear para seleccionar el nivel de flexibilidad adecuado como el grado del polinomio o el nmero de K-Nearest-Neighbors.

En la ejecucin de un modelo al utilizar 100% de sus datos podra generar un resultado que no se generaliza bien a los nuevos datos entrantes. Tambien existe otros escenario, dividir los datos originales en un conjunto de entrenamiento, que generalmente toma alrededor del 70% de los datos y un conjunto de prueba que comprende el resto, el 30%. Se libera el modelo con los datos de entrenamiento y luego usa el conjunto de pruebas para verificar cul es la salida del modelo en comparacin con las respuestas que tienes disponibles.

Sin embargo, este proceso de entrenamiento y prueba an es algo limitado. En una capacidad, cuando est probando la salida del modelo contra los datos reservados, solo est viendo cul es el error para esa agrupacin exacta de los datos de prueba. En teora, los datos de la prueba deberan ser representativos de todo el conjunto de datos, pero en la prctica hay casos en los que eso podra no ser cierto. Se debera entrenar el modelo de tal manera que pueda estar seguro de que el error es representativo de todo el conjunto de datos, no solo del segmento especfico que obtiene de los datos seleccionados al azar que coloca en el conjunto de prueba.

La validacin cruzada es una tcnica estadstica mediante la cual toma todo su conjunto de datos, lo divide en una serie de pequeos fragmentos de entrenamiento/prueba, evala el error para cada fragmento y luego promedia esos errores finales. Este enfoque termina siendo una forma ms precisa de evaluar si su enfoque de modelado tiene algn problema que podra estar oculto en varias combinaciones de las partes de entrenamiento y prueba del conjunto de datos.

De hecho, ya hemos realizado una forma de validacin cruzada! La simple divisin 70/30 de entrenamiento/prueba que es un tipo de "validacin cruzada". Sin embargo, existen muchas otras tcnicas estadsticas de validacin cruzada, y con R teniendo su base en el diseo estadstico, puede modelar muchos tipos diferentes de validacin cruzada.

Tipos de Data Set¶

  • Training data set: observaciones con las que se genera el modelo.

  • Test data set: observaciones del training data set no empleados en la creacin del modelo.

Tipos de errores¶

  • Training error rate: error que comete el modelo al predecir con observaciones del training data set.

  • Test error rate: error que comete el modelo al predecir con observaciones del test data set.

Entonces podemos decir que Cross-Validation o validacin cruzada, se emplea para estimar el test error rate de un modelo y as evaluar su capacidad predictiva. Tambin se puede emplear para seleccionar el nivel de flexibilidad adecuado como grado del polinomio o el nmero de K-Nearest-Neighbors.

Por lo general, en variables cuantitativas, el error se mide mediante mean square error (MSE) o con el RMSE. Cuando se trata de variables cualitativas (clasificacin), el error se mide como la proporcin de predicciones incorrectas respecto al total de predicciones.

Cross-Validation Simple¶

El mtodo ms sencillo consiste en dividir aleatoriamente las observaciones disponibles en dos grupos, uno se emplea para entrenar al modelo y otro para evaluarlo.

LIMITACIONES:

VARIANZA ALTA: La estimacin del test error rate es altamente variable porque depende de qu observaciones se incluyan como set de entrenamiento y cules como set de validacin.

BIAS ALTO: Esto quiere decir que al tener pocos datos para entrenar el modelo, se dispone de menos informacin, por lo que se reduce su capacidad predictiva. Lo que provocara sobrestimacin del test error.

Al tomar muestras de datos, debe tener cuidado con los cuatro niveles diferentes de sesgo y varianza que podra tener

  1. Bajo sesgo(bias), baja varianza: El mejor de los casos. Las muestras son bastante representativas de la poblacin.
  1. Alto sesgo(bias), baja varianza: Todas las muestras son bastante consistentes, pero no reflejan realmente a la poblacin.
  1. Bajo sesgo(bias), alta varianza: Las muestras varan enormemente en su consistencia, pero algunas pueden ser representativas de la poblacin.
  1. Alto sesgo(bias), alta varianza:Las muestras son un poco ms consistentes, pero no es probable que sean representativas de la poblacin.

Cross Validation simple con 100 repeticiones para el calculo del Test error rate

Como se ha comentado previamente, una de las principales desventajas de cross-validation simple es que la estimacin de error vara mucho dependiendo de cmo se hayan repartido los datos entre set de entrenamiento y set de validacin. A continuacin se calcula el test error rate para 100 repeticiones de la validacin, repartiendo en cada una de ellas los datos de forma aleatoria.

In [12]:
cv_MSE <- rep(NA,100)

for (i in 1:100) {

  data.samples <- sample(1:nrow(data), nrow(data) * 0.7, replace = FALSE)
                         
  training.data <- data[data.samples, ]
  
  test.data <- data[-data.samples, ]
  
  train.linear <- lm(y ~ x, training.data)
  
  predicciones <- predict(train.linear, test.data)
  
  cv_MSE[i] <- mean((predicciones  - test.data$y)^2)
}

Flexibilidad del modelo mediante Cross_Validation Simpl: eleccin del grado polinomio del modelo

Introduciendo flexibilidad en el modelo mediante polinomios de mayor grado podra reducir el error de prediccin. La validacin cruzada nos permite identificar con qu grado de polinomio se consigue el mejor modelo (menor test cv-MSE).

A continuacin se ajustan 10 modelos distintos empleando polinomios de grado 1 hasta 10 y se registra el cv-MSE para cada uno. La funcin poly() permite determinar el grado de polinomio empleado.

K-Fold Cross-Validation¶

El mtodo K-Fold Cross-Validation es tambin un proceso iterativo. Consiste en dividir los datos de forma aleatoria en k grupos de aproximadamente el mismo tamao. k-1 grupos se emplean para entrenar el modelo y uno de los grupos se emplea como test, este proceso se repite k veces utilizando un grupo distinto como test en cada iteracin. El proceso genera k estimaciones del test error cuyo promedio se emplea como estimacin final.

Observaciones:

El nmero de iteraciones necesarias viene determinado por el valor k escogido. Por lo general se recomienda un k entre 5 y 10.

La principal ventaja de K-fold CV es que consigue una estimacin precisa del test error rate gracias a un mejor Balance entre bias y varianza. En el mtodo K-fold CV los k grupos empleados como entrenamiento son mucho menos solapantes, lo que se traduce en menor varianza al promediar las estimaciones de error.

En R se puede realizar CV de cualquier generalized linear model creado mediante glm() empleando la funcin cv.glm() del paquete boot. La funcin glm() engloba diferentes tipos de modelos lineales que se especifican mediante el argumento family. En el caso de regresin lineal no se especifica ningn tipo de familia y es equivalente a emplear la funcin lm(). Para regresin logstica se emplea family=binomial.

La funcin cv.glm() calcula el error de prediccin mediante cross-validation. Si el argumento k no se especifica, se le asigna automticamente el nmero de observaciones empleadas para crear el modelo, por lo que se realiza un leave-one-out cross-validation (es decir una 1 observacion para test y el resto para entramiento). La funcin devuelve una lista con mltiples componentes, el resultado de la validacin se almacena dentro del vector delta que contiene la estimacin de error con y sin correccin.
